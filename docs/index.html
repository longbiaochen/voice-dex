ChatType v0.1.2

按住 F5 说话，松开就把中文直接回填到当前输入框。

ChatType 是一个原生 macOS 听写工具，面向已经在这台 Mac 上登录 ChatGPT Desktop 的用户。 它走本地桌面登录态，不要求额外 API key，不要求本地模型下载，核心目标就是把 F5 -> speak -> paste 这条链路做到足够快、足够稳。

下载 v0.1.2 查看 GitHub 仓库 支持继续维护

产品承诺

不额外订阅，不默认要 API key，不额外跑第二轮 AI 清洗，结果优先直接可用。

这次更新

支持从 TypeWhisper 手动导入术语表，并在 STT 之后做更强的本地术语对齐，不引入第二轮 AI 清洗。

安装要求

必须把打包后的 ChatType.app 安装到 /Applications，不要直接从 dist 运行。

使用流程

  1. 下载并安装 ChatType.app 到 /Applications/ChatType.app。
  2. 确保这台 Mac 上已经安装并登录了 Codex Desktop / ChatGPT Desktop。
  3. 首次运行时授予麦克风权限。
  4. 如果你要自动回填到当前输入框，授予 Accessibility 权限。
  5. 把光标放到 Notes、Mail、Slack、Codex 等可输入位置。
  6. 按一次 F5 开始录音，再按一次 F5 结束录音并回填。

权限说明

  • 麦克风：用于录音；首用时会触发系统权限提示。
  • Accessibility：用于把结果回填到当前输入光标；只有检测到可编辑目标时才会报告粘贴成功，否则保留在剪贴板供手动粘贴。
  • 重要：权限修复和验证都必须基于 /Applications/ChatType.app，不要基于 dist/ChatType.app。

v0.1.2 Highlights

  • 设置页新增 Import from TypeWhisper，导入成功后持久化到 ChatType 自己的配置里。
  • 术语对齐仍然是本地确定性后处理，不把产品路径改回第二次模型清洗。
  • 默认 public path 仍然是本机已登录 Codex Desktop + F5 -> speak -> paste。
  • 剪贴板默认保留最后一次转写结果，光标丢失时可以直接手动粘贴；安装路径仍然必须是 /Applications/ChatType.app。

开源与支持

ChatType 是 MIT 许可的开源项目。赞助入口只用于支持后续维护，不改变免费使用路径； 如果你只是想试用或反馈问题，直接从 GitHub Release 下载即可。

ChatType 依赖本地桌面登录态和上游私有转写链路；它是面向个人使用效率的工具，不把这条集成表述成稳定公开 API。
