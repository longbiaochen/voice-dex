# ChatType 7-Day Chinese Promotion Calendar

Goal: low-cost broad awareness first, then developer validation.

Do not post every channel at the same moment. Each post should point to the same release and landing page, but use platform-native framing.

- Platform: Xiaohongshu
- Asset: `exports/xhs-cover-pain-point.jpg`, or source card `assets/xhs-cover-pain-point.svg`
- Copy: `Xiaohongshu Note 1` in `launch-copy.md`

## Day 2: Jike Builder Note

- Platform: Jike
- Copy: `Jike` in `launch-copy.md`

## Day 4: V2EX Developer Validation

- Platform: V2EX
- Node: `分享创造`
- Copy: `V2EX` in `launch-copy.md`

## Day 5: Zhihu / Juejin Long Form

- Platform: Zhihu or Juejin first; mirror to the other after feedback
- Copy: `long-form-article.md`, then adapt platform opening and ending
- CTA: invite users who already use macOS + ChatGPT/Codex Desktop to test
- Track: comments, collections, GitHub referral traffic if available

## Day 7: Bilibili Short Demo

- Format: 30-45 second screen recording
- Script:
  1. "我经常在 Codex 里写长中文需求。"
  2. "现在把光标放在输入框，按 F5。"
  3. "说完再按 F5。"
  4. "能粘贴就直接回填，不能粘贴就留在剪贴板。"
  5. "项目开源，面向已登录 ChatGPT/Codex Desktop 的 Mac 用户。"
- Title:
  > 给 Mac + ChatGPT/Codex 做了一个 F5 语音输入工具
- Track: completion rate, comments, GitHub visits

## After Day 7

- If there are 3-5 real feedback items, prepare Show HN.
- If GitHub stars and installs move, submit AI directories.
- If repeated questions focus on trust, add a landing-page FAQ before Product Hunt.
